± Git status symbols — the lingua franca

Inside (branch …). Starship's defaults, shared with many prompts — worth memorizing. Each shows only when present; several stack, e.g. ✘!?.

Working tree

!modifiedtracked file edited, not staged
+stagedchanges added with git add
?untrackednew file git isn't tracking yet
✘deleteda tracked file was removed
»renameda tracked file was renamed
$stashedyou have entries in git stash
=conflictedmerge/rebase conflict to resolve

Vs. upstream

⇡NaheadN local commits not yet pushed
⇣NbehindN upstream commits not yet pulled
⇕divergedboth ahead and behind — needs a rebase
Nothing after the branch = clean tree. So (master ✘!?) reads as: on master, with a deletion, an unstaged edit, and an untracked file. All symbols are remappable under [git_status].

◆ Attach & detach — the whole point

A session keeps running on the machine after you disconnect. Reconnect and pick up exactly where you left off.

Start / attach a session named main
ta⭐ortmux new -A -s main
Start a named sessione.g. one per agent
tmux new -s agent
Detach — leave it running
C-Spcthend
List sessions
tl⭐ortmux ls
Attach to a session
tmux attach -t agent
Kill a session
tmux kill-session -t agent
Reconnecting over SSH: just ssh in normally and run ta. Doing it from an interactive login refreshes your SSH-agent link, so existing panes keep using your forwarded keys. Avoid ssh host -t tmux attach — it's non-interactive and skips that refresh.

◨ cw — two-pane Claude workspace ⭐

A shell helper that opens a new window (current pane untouched) split into two side-by-side panes rooted in a directory: left runs Claude Code via cswap run, right is a plain shell. Focus lands on the Claude pane.

New window hereClaude via the dir's cswap-mapped account
cw
…forcing a cswap accountoverrides the mapping
cw 2
…account 2, rooted in a directory
cw 2 ~/src/foo
Mapped account, rooted in a directoryfirst arg is the dir when it looks like a path
cw ~/src/foo
Omitting the profile uses the directory's cswap map account — so map your projects once (cswap map 2 ~/src/foo) and a bare cw there just works; pass a number to override. Must be run from inside tmux. Defined in source/50_tmux.sh.

↺ Persist across reboots ⭐

Detach/reattach needs no help. These add recovery after a reboot or tmux-server restart (resurrect + continuum).

Save layout now
C-SpcC-s
Restore last saved layout
C-SpcC-r
Autosave + restore on startupcontinuum, every 15 min — automatic
on by default
A reboot kills running processes (agents too). These restore the layout, panes, and cwd so you can relaunch fast — they don't preserve a live process's in-memory state.

$ tmux from the plain shell

No prefix needed — these run before you're inside tmux.

# sessions
tmux ls                      # list sessions
tmux new -s NAME             # create a session
tmux attach -t NAME          # attach to one
tmux kill-session -t NAME    # kill one session
tmux kill-server             # kill everything

tmux source-file ~/.tmux.conf   # reload config from outside tmux
